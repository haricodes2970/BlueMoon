# Security Policy

## Reporting a Vulnerability

BlueMoon is a communication platform; security issues are treated with high
priority. If you discover a vulnerability, do not open a public issue.

Instead, report it privately to the maintainers with:

- A description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested remediation

Reports will be acknowledged and triaged as soon as possible. Once a fix is
available, a disclosure timeline will be coordinated with the reporter.

## Scope

This policy covers the BlueMoon repository and any deployed BlueMoon/PINChat
services. Third-party dependencies should be reported to their respective
maintainers, though we appreciate being notified if BlueMoon is affected.

## Security Documentation

Ongoing security architecture, threat modeling, and review notes are
maintained under [`docs/security`](./docs/security).
